# 2026-07-15 XTheme 会话记忆

- 用户反馈其他用户在线安装最新版后，重启仍然出现 `/boot/config/plugins-error/xtheme.plg`，主题丢失。
- 排查判断核心风险是发布版 `.plg` 在每次开机安装时仍通过 `<FILE><URL>` 强制从 GitHub 下载 `.b64` 包；Unraid 启动阶段网络、DNS 或 GitHub 访问不稳定时会失败并把插件移动到 `plugins-error`。
- 修复方向：发布版 `.plg` 改为优先使用 `/boot/config/plugins/xtheme/packages/` 内的本地缓存 `.txz` 包，只有缓存缺失或 MD5 不匹配时才下载 `.b64`，下载后解码成 `.txz` 并保留给后续重启使用。
- 本地验证版本为 `2026.07.15.1`。
- 已在用户 Unraid 测试机上用本地包升级成功，并模拟开机路径：删除 `/var/log/plugins/xtheme.plg` 和 `/usr/local/emhttp/plugins/xtheme` 后，仅依赖 `/boot/config/plugins/xtheme.plg` 与本地缓存包重装成功，`plugins-error` 未出现。
